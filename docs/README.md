# OctoAcme Project Management Docs

## Welcome

This directory contains the complete OctoAcme project management framework—a set of lightweight, practical guides for managing cross-functional projects from initiation through close.

## 📖 Overview of OctoAcme Project Management

OctoAcme follows a structured five-phase project lifecycle designed to deliver customer value iteratively while maintaining clear ownership, transparency, and continuous improvement. The framework encompasses **Initiation** (validating business needs and stakeholder alignment), **Planning** (breaking work into prioritized backlogs with clear acceptance criteria), **Execution** (day-to-day delivery with regular demos and quality gates), **Release** (standardized deployment with safety checks), and **Retrospective** (capturing learnings for continuous improvement).

The framework is built on five core principles: **customer-first** (prioritizing value and usability), **iterative delivery** (small, testable increments), **clear ownership** (named PM and Product Lead per project), **data-informed decisions** (measuring impact and iterating based on evidence), and **psychological safety** (encouraging feedback and learning). 

Communication is structured through regular cadences: daily standups, weekly PM-PdM syncs, twice-weekly delivery team standups, and monthly stakeholder updates. Risks are managed proactively through a Risk Register and three-level escalation paths, ensuring blockers surface early. Quality is embedded at every stage through Definition of Done standards, automated CI/CD, testing checkpoints, and security scanning—enabling reliable, maintainable code delivery.

Three core roles drive execution: **Project Managers** coordinate delivery and manage risks; **Product Managers** define priorities and success metrics; and **Developers** implement features and validate solutions. This shared responsibility model ensures accountability and reduces ambiguity.

- **Customer-first**: Prioritize customer value and usability.
- **Iterative delivery**: Deliver small, testable increments.
- **Clear ownership**: Each project has a named PM and Product Lead.
- **Data-informed**: Measure impact and iterate based on evidence.
- **Psychological safety**: Encourage feedback and learning.
